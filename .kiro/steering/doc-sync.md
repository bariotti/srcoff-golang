# Regra: Manter Documentação Sincronizada

Após qualquer alteração de código, implementação de tarefa, ou resposta a um prompt do usuário, você DEVE atualizar os seguintes documentos se houver mudanças relevantes:

1. `.kiro/specs/{feature-name}/requirements.md` — atualizar se requisitos mudaram ou foram refinados
2. `.kiro/specs/{feature-name}/design.md` — atualizar se a arquitetura ou design mudou
3. `.kiro/specs/{feature-name}/tasks.md` — marcar tarefas concluídas e adicionar novas se necessário
4. `README.md` — manter atualizado com instruções de uso, estrutura do projeto e status atual

Esta regra se aplica a TODAS as interações, sem exceção.
